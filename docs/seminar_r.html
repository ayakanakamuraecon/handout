01Posit Cloud のセットアップ

▼ Posit Cloud のセットアップ（初回のみ）

Posit Cloud は、ブラウザ上で R を使えるサービスです。インストール不要で、どの PC からでも同じ環境で作業できます。

  • インストール不要・ブラウザで完結
  • 作業がクラウドに自動保存される
  • 授業の PC でも自宅の PC でも続きから作業できる
注意：無料プランで十分使えますが、月25時間の制限があります。授業以外で使いすぎないよう注意してください。自分の PC に RStudio をインストールすれば時間制限なく使えます。

準備するもの

  • PC（Windows または Mac）※スマホ・タブレット不可
  • Google アカウント
  • ブラウザは Chrome または Edge を推奨
1
Posit Cloud にアクセスブラウザで posit.cloud を開き、「Get Started for Free」をクリック
2
アカウントを作成「Sign Up with Google」→ Google アカウントでログイン → 利用規約に同意（すでにアカウントがある人は「Log In」から）
3
プロジェクトを作成右上「New Project」→「New RStudio Project」→ 名前を入力（例：seminar2026）して Enter
4
tidyverse をインストール左下の Console に下記を貼り付けて Enter
5
動作を確認続けて確認用コードを貼り付けて Enter。グラフが表示されれば完了

手順4：Console に貼り付けるコード

install.packages("tidyverse")
補足
インストールには1〜2分かかります。赤い文字が出ることがありますが、エラーでなければ問題ありません。

手順5：続けて貼り付けるコード

library(tidyverse) ggplot(mpg, aes(x = displ, y = hwy)) + geom_point()

02画面の見方

▼ 画面の見方

RStudio の画面は4つの領域に分かれています。

Source（左上）

コードを書く場所。コードはファイルとして保存できる

Environment（右上）

作成したオブジェクト（データや変数）の一覧

Console（左下）

コードが実行され、結果が表示される場所。ここに直接コードを書くことも可能

Files / Plots / Packages / Help（右下）

ファイル一覧、作成したグラフ、パッケージ、ヘルプ。使うファイルや作成したファイルはここから確認する

コードの実行方法：Source に書いたコードは Ctrl + Enter（Mac は Cmd + Enter）で実行します。1行だけ実行したいときはその行にカーソルを置き、複数行実行したいときは範囲を選択してから押します。
補足：Source と Console の使い分け
Console は試し打ち用で、入力した内容は残りません。あとから見返したいコードは Source に書いて保存してください。

03R の基本操作

パッケージ

R の機能を拡張する道具をパッケージといいます。tidyverse は、データ操作と可視化のための代表的なパッケージです。

install.packages("tidyverse") インストール（初回の1回だけ）
library(tidyverse) 読み込み（作業のたびに毎回）
補足
新しいセッションを開いたときは、library() を実行しないと関数が使えません。「関数が見つからない」というエラーが出たら、まず library() を実行したか確認してください。

基本演算

1 + 1 # 足し算 10 - 3 # 引き算 4 * 5 # 掛け算 20 / 4 # 割り算 2^3 # べき乗（2の3乗） 10 %% 3 # 余り（10 ÷ 3 の余り）
結果
[1] 2
[1] 7
[1] 20
[1] 5
[1] 8
[1] 1

オブジェクト（変数への代入）

<- を使うと、値をオブジェクトに保存できます。保存した値は、あとから何度でも使えます。

x <- 10 # xに10を代入 y <- 3 # yに3を代入 x + y # 計算できる z <- x * y # 計算結果を新しいオブジェクトに保存 z # 中身を確認
結果
[1] 13
[1] 30
補足
代入しただけでは何も表示されません。中身を見たいときは、オブジェクト名だけを実行します。

ベクトル

複数の値をまとめたものをベクトルといいます。c() で作成します。

scores <- c(80, 65, 90, 72, 88) length(scores) # 要素数 sum(scores) # 合計 mean(scores) # 平均 max(scores) # 最大値 min(scores) # 最小値
結果
[1] 5
[1] 395
[1] 79
[1] 90
[1] 65

要素の取り出し

scores[1] # 1番目 scores[2:4] # 2〜4番目
結果
[1] 80
[1] 65 90 72
補足
要素の番号は0ではなく1から始まります。

05データの前処理

パイプ演算子

%>%（パイプ演算子）は、左の結果を右の関数に渡す記号です。処理を順番につなげて書けるため、コードが読みやすくなります。

df %>% filter(year == 2008) # df というデータを filter() に渡して、2008年の行だけを取り出す

filter：条件に合う行を取り出す

df %>% filter(year == 2008) # 2008年のデータのみ df %>% filter(cyl >= 6) # 気筒数6以上 df %>% filter(manufacturer == "toyota") # トヨタのみ
補足
「等しい」の条件はイコール2つ（==）です。1つ（=）は代入の意味になるためエラーになります。文字列は必ず " で囲みます。

select：列（変数）を選ぶ

df %>% select(manufacturer, model, year, hwy)

mutate：新しい変数を作る

df %>% mutate(hwy_km = hwy * 1.609) # マイル→kmに変換した新しい列を追加

summarise：集計する

df %>% summarise( mean.hwy = mean(hwy), max.hwy = max(hwy), n = n() )
補足
n() はデータの件数を数える関数です。

group_by：グループ別に集計する

df %>% group_by(manufacturer) %>% summarise(mean.hwy = mean(hwy)) %>% arrange(desc(mean.hwy)) # 降順に並び替え
補足
arrange() は並び替えの関数です。デフォルトは昇順で、desc() で囲むと降順になります。

Word形式で保存する

datasummary(displ + hwy + cty ~ Mean + SD + Min + Max + N, data = df, output = "flextable") %>% save_as_docx(path = "descriptive_stats.docx")

output = "flextable" を付けて save_as_docx() に渡す、という形は、後述の回帰結果表の保存とまったく同じです。この型を覚えておけば、どちらの表も Word に出力できます。

07可視化

グラフの作成には ggplot2（tidyverse に含まれる）を使います。

ggplot2 のグラフは、透明なシートを1枚ずつ重ねてグラフを完成させるようなイメージで組み立てます。

第1層：データ
ggplot(df, ...) — どのデータを使うか
＋
第2層：軸
aes(x = displ, y = hwy) — どの変数を軸に置くか
＋
第3層：グラフの種類
geom_point() — どんなグラフにするか
  • ①ggplot(df, ...) — 描画画面を用意する
  • ②+ aes(x = displ, y = hwy) — 軸を追加する
  • ③+ geom_point() — グラフ（点）を描き足す

ヒストグラム（1変数の分布）

ggplot(df, aes(x = hwy)) + geom_histogram(binwidth = 2)
補足
binwidth は棒の幅。データに合わせて調整します。

散布図（2変数の関係）

ggplot(df, aes(x = displ, y = hwy)) + geom_point()

棒グラフ（グループ別の平均）

df %>% group_by(class) %>% summarise(mean_hwy = mean(hwy)) %>% ggplot(aes(x = class, y = mean_hwy)) + geom_bar(stat = "identity")
補足
geom_bar() のデフォルトは stat = "count" で、カテゴリごとの件数を数えて棒の高さにします。上のようにあらかじめ summarise() で集計した値をそのまま棒の高さに使いたいときは、stat = "identity" を指定します。

箱ひげ図（グループ別の分布）

ggplot(df, aes(x = class, y = hwy)) + geom_boxplot()

グラフの保存

ggsave("my_plot.png") # 最後に表示したグラフを保存 p <- ggplot(df, aes(x = displ, y = hwy)) + # 名前を付けて保存 geom_point() ggsave("my_plot.png", plot = p)
補足
width（幅）・height（高さ）・dpi（解像度）も指定できます。
ggsave("my_plot.png", plot = p, width = 8, height = 5, dpi = 150)
補足
保存したファイルは右下の Files タブに表示されます。ダウンロードするには、Files タブでファイルにチェックを入れ、More → Export を選びます。

08回帰分析

単回帰分析

lm() で回帰式を推定します。Y ~ X の形で書き、data = に使うデータを指定します。

model1 <- lm(hwy ~ displ, data = df) # モデルの推定 summary(model1) # 結果の確認
結果
Coefficients:
            Estimate Std. Error t value Pr(>|t|)
(Intercept)  35.6977     0.7204   49.55   <2e-16 ***
displ        -3.5306     0.1945  -18.15   <2e-16 ***

Residual standard error: 3.836 on 232 degrees of freedom
Multiple R-squared:  0.5871,    Adjusted R-squared:  0.5848
F-statistic: 329.9 on 1 and 232 DF,  p-value: < 2.2e-16
結果の読み方
  • Estimate（係数）— X が1増えると Y がどれだけ変化するか
  • Pr(>|t|)（p値）— 一般的には0.05未満なら統計的に有意
  • R-squared — モデルの当てはまりの良さ（0〜1）

詳しい解釈は「分析と解釈」のページを参照してください。

重回帰分析

説明変数を増やすときは + でつなぎます。

model2 <- lm(hwy ~ displ + cyl, data = df) model3 <- lm(hwy ~ displ + cyl + factor(year), data = df) summary(model2) summary(model3)
補足
factor() で変数をカテゴリ型に変換すると、lm() が自動的にダミー変数に変換して回帰式に組み込みます。年など、数値としてではなくカテゴリとして扱いたい変数に使います。

結果表の作成

modelsummary() を使うと、複数のモデルを並べた結果表を作れます。

補足
記述統計で読み込んだ modelsummary と flextable をそのまま使います。セッションを開き直した場合は library() を再実行してください。
modelsummary( list("モデル1" = model1, "モデル2" = model2, "モデル3" = model3), stars = TRUE, # 有意水準の星印を表示 coef_omit = "factor", # factor() で作ったダミー変数を出力しない gof_omit = "IC|Log|F" # 不要な適合度指標を省略 )
結果
              モデル1     モデル2     モデル3
displ         -3.531***   -2.221***   -2.152***
              (0.194)     (0.372)     (0.373)
cyl                       -1.246***   -1.192***
                          (0.301)     (0.300)
Num.Obs.       234         234         234
R2             0.587       0.610       0.613
補足
msummary() という書き方も見かけますが、modelsummary() と同じ関数です。

結果表をWord・PowerPointで保存する

modelsummary(model1, output = "flextable") %>% save_as_docx(path = "regression_table.docx") modelsummary(model1, output = "flextable") %>% save_as_pptx(path = "regression_table.pptx")

output = "regression_table.docx" と直接ファイル名を書く方法は、modelsummary v2.0以降では動きません。上記のように output = "flextable" と save_as_docx() を組み合わせてください。

09エラーと困ったときは

「Error」と赤い文字が出た

エラーはよくあることです。まずエラー文をコピーして Teams の質問チャンネルに投稿してください。コードの打ち間違い（スペース、記号、大文字小文字）が原因のことが多いので、まず見直してみましょう。

「could not find function」と出る

その関数を含むパッケージが読み込まれていません。library() を実行してください。

「object not found」と出る

指定したオブジェクトがまだ作られていないか、名前が違います。Environment タブで確認してください。

インストールが終わらない

時間がかかることがあります。5分以上動きがなければ、ページを更新して再度試してください。

プロジェクトはどこに保存される？

Posit Cloud のサーバーに自動保存されます。「Save」を押す必要はありません。次回ログインすれば同じプロジェクトから再開できます。

自宅の PC でも使える？

使えます。同じアカウントでログインすれば、続きから作業できます。

困ったときは：Teams の「質問チャンネル」に投稿する（スクリーンショット添付推奨）、または授業の前後に教員に直接聞いてください。
