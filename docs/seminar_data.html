02前処理の一般的な工程

番号付き（1→2→3）は順番どおりに行う工程です。点（・）はどの順で行ってもよい工程です。

1

準備

  1. Posit Cloud で研究用プロジェクトを作成する
  2. データファイル（CSV等）をプロジェクトにアップロードする
  3. アップロードしたファイルが Files パネルに表示されているか確認する
2

読み込み

  1. read_csv("ファイル名.csv") を実行する
  2. エラーが出ずに読み込めるか確認する（文字コードエラーが出たら locale を指定）
  3. 読み込んだデータをオブジェクトに代入する（df <- read_csv(...)）
3

構造確認

  • head(df) で先頭数行、各列の型、行数・列数の見当を確認する
  • summary(df) で基本統計量を確認する
4

問題発見

  • 列名が日本語・スペース入りでないか確認する
  • 数値のはずの列が chr になっていないか確認する
  • 欠損値があるか確認する（is.na(df) %>% colSums()）
  • 横持ちになっていないか確認する（年次等が列名に入っていないか）
5

整形

  • 列名を変更する（rename()）
  • 型を変換する（mutate() と as.numeric() など）
  • 横持ちなら pivot_longer() で縦持ちに変換する
  • 複数ファイルがあれば left_join() で結合する
  • 不要な列・行を取り除く（select()・filter()）

整形フェーズの注意：不要な列・行は、後の工程で使わないと確信できた時点で早めに落とすと効率的です。ただし、left_join() の結合キーや filter() の条件に使う列は、その処理より前に落とさないよう注意してください。

03工程の詳細解説

▼ 型の変換

read_csv() でデータを読み込むと、各列に型が割り当てられます。型が合っていないと集計・分析でエラーが出るので確認が必要です。

  • dbl・int：数値である。そのまま計算できる
  • chr：文字列である。数値のように見えても計算できない
  • fct：因子型である。カテゴリ変数に使う

型を確認するには head(df) を使います。数値のはずの列が chr になっている場合は、次のように変換します。

df %>% mutate(変数名 = as.numeric(変数名))
▼ 欠損値の扱い

データに欠損値（NA）が含まれていると、集計や分析の結果に影響します。まず、どの変数にどれだけ欠損があるかを確認し、除外するか補完するかを判断してください。欠損が多い変数は、そもそも分析に使えない可能性があります。

is.na(df) %>% colSums() # 各変数の欠損数を確認 df %>% filter(is.na(変数名)) # 欠損のある行を表示 df %>% drop_na(変数名) # 欠損のある行を除外
▼ 横持ち→縦持ちの変換

e-Stat などでよくある横持ちデータは、pivot_longer() で整形できます。

df %>% pivot_longer( cols = c("2019年", "2020年", "2021年"), # 縦に変換したい列 names_to = "year", # 列名を格納する新しい変数名 values_to = "productivity" # 値を格納する新しい変数名 )
